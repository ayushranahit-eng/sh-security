# Architecture And Flow

## Main Files

| File | What It Does |
| --- | --- |
| `scan.sh` | Main Bash script run by the client |
| `find_exposed_files.sh` | Finds risky files like `.env`, `.git`, `.sql`, private keys |
| `merge_report.py` | Combines all scanner outputs into one JSON report |
| `server.py` | Serves rule files and optionally receives uploaded reports |
| `dist/semgrep-rules.yml` | Source-code rules |
| `dist/iac-rules.yml` | Docker/IaC/CI/CD/config rules |
| `sample-vulnerable-repo/` | Fake unsafe repo for testing |

## Simple Flow

```text
1. Client runs scan.sh inside their codebase.
2. scan.sh downloads or loads scanner rules.
3. scan.sh runs available checks:
   - custom file checks
   - Semgrep source-code rules
   - Semgrep IaC/config rules
   - gitleaks, if available
   - osv-scanner, if available
4. Raw outputs are saved in .scan-sh/
5. merge_report.py creates one final JSON report.
6. Optional: report is uploaded to server.py if SCAN_API_TOKEN is set.
```

## Report Output

The final report filename includes date and time:

```text
security-report-YYYYMMDD-HHMMSS.json
```

Example:

```text
security-report-20260612-161114.json
```

## Temporary Scanner Folder: `.scan-sh/`

When `scan.sh` runs, it creates a temporary working folder inside the scanned project:

```text
.scan-sh/
```

This folder is not part of the application source code.

It is generated by the scanner.

It stores raw outputs from each scanner step:

```text
.scan-sh/custom.json
.scan-sh/semgrep-sast.json
.scan-sh/semgrep-iac.json
.scan-sh/gitleaks.json
.scan-sh/osv.json
.scan-sh/*.err
.scan-sh/*.status.json
```

Then `merge_report.py` reads these raw files and creates the final report:

```text
security-report-YYYYMMDD-HHMMSS.json
```

If `.scan-sh/` is deleted, the next scan will create it again.

It is regenerated locally. It is not downloaded from the server.

Recommended `.gitignore` entries:

```text
.scan-sh/
security-report*.json
```

To delete `.scan-sh/` automatically after report creation:

```bash
./scan.sh --clean
```

or:

```bash
SCAN_CLEAN=1 ./scan.sh
```

## Local Server Role

The local server is only needed for testing or centralized rule/report handling.

It provides:

```text
GET /dist/semgrep-rules.yml
GET /dist/iac-rules.yml
POST /api/report
GET /api/report/{id}
```

The scanner can also run offline if tools and rules are already present.
